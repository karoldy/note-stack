# Rspress Container (Callout) 语法参考

为重要信息添加视觉标注，6 种内置类型，语法为 `:::type` 包裹。

## 基本语法

```markdown
:::tip
这是一条提示信息。
:::

:::warning
这是一条警告信息。
:::

:::danger
这是一条危险警告。
:::
```

## 六种类型

| 类型 | 渲染效果 | 适用场景 |
| --- | --- | --- |
| `note` | 灰色边框 📝 | 中性补充说明 |
| `tip` | 绿色边框 💡 | 最佳实践、技巧建议 |
| `info` | 蓝色边框 ℹ️ | 参考信息、外部链接 |
| `warning` | 黄色边框 ⚠️ | 注意事项、废弃提醒 |
| `danger` | 红色边框 🚫 | 危险操作、禁止行为 |
| `details` | 可折叠区块 | 冗长附加内容、折叠详情 |

## 自定义标题

### 内联写法（推荐）

```markdown
:::tip 默认模型选择
除非用户明确指定，始终使用 `claude-opus-4-8`。
:::
```

### 大括号写法（MDX 文件需注意转义）

```markdown
:::tip{title="默认模型选择"}
除非用户明确指定，始终使用 `claude-opus-4-8`。
:::
```

> **建议**：优先使用内联写法，大括号写法在 `.mdx` 文件中需要额外转义处理。

## GitHub Alerts 语法

Rspress 同时支持 GitHub 风格的 blockquote 写法，标签内使用**大写**：

```markdown
> [!NOTE]
> 这是一条笔记。

> [!WARNING]
> 这是一条警告。

> [!TIP]
> 这是一条提示。
```

| GitHub 语法 | 等价 `:::` |
| --- | --- |
| `> [!NOTE]` | `:::note` |
| `> [!TIP]` | `:::tip` |
| `> [!INFO]` | `:::info` |
| `> [!WARNING]` | `:::warning` |
| `> [!DANGER]` | `:::danger` |

## 重要规则

1. **类型名必须小写**：`:::tip` 正确，`:::Tip` 不会被识别
2. **GitHub Alerts 标签必须大写**：`[!NOTE]` 正确，`[!note]` 不会被识别
3. 两种语法渲染结果相同，选一种在项目内统一使用
4. `details` 类型创建可折叠区块，适合 FAQ 或冗长的附加说明
5. 容器可以嵌套代码块、列表、表格等任意 Markdown 内容

## 使用场景对照

| 原文写法 | 改进为容器 |
| --- | --- |
| `> **注意**：xxx 已废弃。` | `:::warning 已废弃` |
| `> **不要 xxx**` | `:::danger 禁止 xxx` |
| `> 提示：可以 xxx` | `:::tip` |
| 无标注的补充段落 | `:::note` |
| 冗长的配置示例 | `:::details 配置详情` |
